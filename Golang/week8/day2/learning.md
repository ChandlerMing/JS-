# Week8 Day2 学习内容（超详细指导）

## 今日主题
- 日志规范

## 学前准备
- 了解时间格式。
- 工作目录：`Golang/week8/day2`
- 先执行：`go run template.go`

## 分钟级安排（建议 120 分钟）
- 0-15 分钟：快速阅读模板，标记看不懂的点
- 15-40 分钟：学习核心知识，写下 3 条笔记
- 40-85 分钟：按步骤改代码并验证输出
- 85-105 分钟：做扩展练习
- 105-120 分钟：复盘和整理提交物

## 核心知识讲解
- 日志最少含时间/级别/消息。
- 结构化字段便于检索。
- 控制日志噪声。

## 手把手实操（命令级）
1. 打开当前目录下 `template.go`
2. 运行：`go run template.go`
3. 按下面子步骤逐条修改并运行
   - 运行模板观察日志格式。
   - 新增 ERROR 级别与错误码。
   - 修改一个核心函数并重新运行验证。
   - 补充一个边界输入并记录输出。
   - 将关键逻辑提取为可复用函数。
4. 每完成一步，至少再运行一次确保无回归
## 常见错误与排查
- 日志无上下文。
- 调试日志混入生产。
- 若报错看不懂，先精简到最小复现场景再定位

## 参考答案要点（不是唯一答案）
- 函数边界清晰：输入/输出/错误语义明确
- 至少覆盖一个边界输入（空、零值、越界、非法）
- 输出内容可解释，不依赖“碰巧正确”

## 自测题（完成后自查）
1. 今天的关键抽象是什么？一句话说明。
2. 如果把某一步删掉，会出现什么错误或风险？
3. 这份实现和 TypeScript 思维最大的差异是什么？
4. 你能给出一个更健壮的版本吗？（例如加入校验/超时/日志）

## 扩展练习
- 定义统一 Logger 接口。

## 完成定义（DoD）
- `go run template.go` 成功
- 你完成至少 1 个扩展点
- 你写了 5-10 行复盘（问题、修复、收获）
- 你能口述今天代码的主流程

## 今日提交物
- 修改后的 `template.go`（或另存 `practice.go`）
- `notes.md`（可选）：记录命令、坑点、结论
