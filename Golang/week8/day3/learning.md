## 核心知识讲解
- `context` 传播超时与取消。
- 下游必须监听 `ctx.Done()`。
- 取消是控制流，不是异常。

## 手把手实操（命令级）
1. 打开当前目录下 `template.go`
2. 运行：`go run template.go`
3. 按下面子步骤逐条修改并运行
   - 运行模板看取消触发。
   - 调整超时做 A/B 对比。
   - 输出取消原因和耗时。
   - 新增重试并传递 ctx。
4. 每完成一步，至少再运行一次确保无回归
## 常见错误与排查
- 创建 context 不 cancel。
- 忽略取消导致泄漏。
- 若报错看不懂，先精简到最小复现场景再定位

## 参考答案要点（不是唯一答案）

- 至少覆盖一个边界输入（空、零值、越界、非法）
- 输出内容可解释，不依赖“碰巧正确”
