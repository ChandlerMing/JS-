# Week5 Day3 学习内容（超详细指导）

## 今日主题
- 统一错误响应

## 学前准备
- 了解 JSON 编码。
- 工作目录：`Golang/week5/day3`
- 先执行：`go run template.go`

## 分钟级安排（建议 120 分钟）
- 0-15 分钟：快速阅读模板，标记看不懂的点
- 15-40 分钟：学习核心知识，写下 3 条笔记
- 40-85 分钟：按步骤改代码并验证输出
- 85-105 分钟：做扩展练习
- 105-120 分钟：复盘和整理提交物

## 核心知识讲解
- 错误结构统一便于前端处理。
- 错误码与错误消息分离。
- 系统错误需隐藏内部细节。

## 手把手实操（命令级）
1. 打开当前目录下 `template.go`
2. 运行：`go run template.go`
3. 按下面子步骤逐条修改并运行
   - 运行模板查看 envelope。
   - 新增 `request_id` 字段。
   - 定义 3 个业务错误码。
   - 把错误输出封装函数。
   - 写一段错误码使用说明。
4. 每完成一步，至少再运行一次确保无回归
## 常见错误与排查
- 同类错误用不同 code。
- 直接暴露内部错误栈。
- 若报错看不懂，先精简到最小复现场景再定位

## 参考答案要点（不是唯一答案）
- 函数边界清晰：输入/输出/错误语义明确
- 至少覆盖一个边界输入（空、零值、越界、非法）
- 输出内容可解释，不依赖“碰巧正确”

## 自测题（完成后自查）
1. 今天的关键抽象是什么？一句话说明。
2. 如果把某一步删掉，会出现什么错误或风险？
3. 这份实现和 TypeScript 思维最大的差异是什么？
4. 你能给出一个更健壮的版本吗？（例如加入校验/超时/日志）
